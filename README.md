# 基于 RK3568 的多传感器布防告警终端

一个基于 **鲁班猫 2（RK3568）** 和 **嵌入式 Linux** 的小型布防告警项目。  
项目以自定义 Linux 字符设备驱动为基础，在用户态实现状态机逻辑，并通过轻量 Web 页面展示系统状态。

整个系统围绕“**环境感知 + 运动检测 + 状态机控制 + 声光告警**”展开，适合作为嵌入式 Linux 驱动和用户态联动的综合练习项目。

---

## 项目简介

本项目主要完成了以下几部分内容：

- 基于 GPIO 实现 **LED / 蜂鸣器输出驱动**
- 基于 GPIO 中断 + `poll` 实现 **按键驱动**
- 基于 I2C 实现 **BH1750 光照传感器驱动**
- 基于 I2C 实现 **MPU6050 六轴传感器驱动**
- 在用户态实现 **布防 / 告警 / 静音 / 空闲** 状态机
- 通过 Web 页面实时展示系统状态、光照值和运动信息

系统在布防状态下，会结合：

- 当前是否处于暗环境
- 是否检测到连续明显运动

来判断是否进入告警状态，并通过 LED / 蜂鸣器进行本地提示。

## 系统架构

整体结构如下：

```text
驱动层
├── alarm_io    -> 控制蓝灯 / 红灯 / 蜂鸣器
├── alarm_key   -> 提供按键事件（支持 poll）
├── bh1750      -> 提供环境光照 lux
└── mpu6050     -> 提供加速度 / 温度 / 角速度原始数据

用户态
└── main.c
    ├── 读取按键事件
    ├── 读取光照与运动数据
    ├── 判断暗环境与运动变化
    ├── 维护状态机
    └── 控制声光输出

展示层
└── Flask + HTML
    └── 实时显示状态、lux、运动信息等
```
## 状态机设计

### 系统包含 4 个核心状态：

IDLE：空闲 / 撤防状态
ARMED：布防状态
ALARM：告警状态
MUTE：静音状态

### 状态流转关系：

IDLE -> ARMED：按键触发
ARMED -> IDLE：按键触发
ARMED -> ALARM：暗环境 + 连续检测到明显运动
ALARM -> MUTE：按键触发
MUTE -> IDLE：按键触发
核心功能
1. GPIO 输出驱动

通过 /dev/alarm_io 控制：

蓝灯
红灯
蜂鸣器

2. 按键中断驱动

通过 /dev/alarm_key 提供按键事件，支持：

中断触发
软件消抖
等待队列
poll 事件通知

3. 光照检测

通过 /dev/bh1750 读取环境光照值（lux），用于判断当前是否处于暗环境。

4. 运动检测

通过 /dev/mpu6050 连续读取一帧原始数据，包括：

三轴加速度
温度
三轴角速度

用户态通过比较前后两帧变化量，判断当前是否发生明显运动。

5. 用户态状态机

在 main.c 中完成：

按键状态切换
暗环境判定
运动检测判定
连续命中计数
告警逻辑控制

6. Web 状态展示

通过 Flask 提供简单 Web 页面，实时展示：

当前状态
光照值
是否暗环境
是否检测到运动
温度
加速度 / 角速度变化量

## 构建与运行

1. 编译驱动

根据内核源码树和 Makefile 进行编译。

2. 加载驱动

按顺序加载：

insmod alarm_io_drv.ko
insmod alarm_key_drv.ko
insmod bh1750_drv.ko
insmod mpu6050_drv.ko

确认设备节点存在：

ls /dev/alarm_io
ls /dev/alarm_key
ls /dev/bh1750
ls /dev/mpu6050

3. 编译用户态程序
gcc main.c -o alarm_app

4. 运行主程序
./alarm_app

5. 启动 Web 页面（可选）
python3 app.py

然后在浏览器访问：

http://开发板IP:5000
